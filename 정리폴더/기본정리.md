\*\*Firebase\*\*를 프로젝트의 핵심 인프라로 도입하면, 보안과 실시간 데이터 관리를 모두 잡은 \*\*"상용화 직전 수준의 SaaS"\*\*를 제안할 수 있습니다. 



최종적으로 확정된 \*\*\[EduChain AI: Firebase 기반 차세대 교육 SaaS 솔루션]\*\*의 전체 기획 및 구현 명세서를 정리해 드립니다.

\### 3. 세부 DB 설계 (Firebase Firestore)



NoSQL 기반의 유연한 구조로 설계하여 실시간 데이터 동기화를 지원합니다.



\*   \*\*`Organizations` (Collection):\*\*

&#x20;   \*   `org\_id`: 기업 코드 (PK)

&#x20;   \*   `org\_name`: 기업/학원명

&#x20;   \*   `max\_slots`: 구매한 학생 수 (예: 100)

&#x20;   \*   `plan`: 요금제 등급 (Starter/Pro/Enterprise)

\*   \*\*`Users` (Collection):\*\*

&#x20;   \*   `uid`: Firebase Auth 고유 ID (PK)

&#x20;   \*   `email`: 이메일

&#x20;   \*   `role`: 사용자 역할 (Teacher/Student/Operator)

&#x20;   \*   `org\_id`: 소속 기업 ID (FK)

\*   \*\*`ChatLogs` (Collection):\*\*

&#x20;   \*   `log\_id`: 기록 ID

&#x20;   \*   `uid`: 학생 ID

&#x20;   \*   `query`: 질문 내용 / `answer`: AI 답변

&#x20;   \*   `timestamp`: 질문 시간

\### 4. 사용자별 주요 기능 및 UI/UX 시나리오



\#### ① 공통: Firebase 로그인 (Gateway)

\*   \*\*UI:\*\* 깔끔한 이메일/비번 로그인 화면.

\*   \*\*로직:\*\* 로그인 성공 시 Firestore에서 `role`을 읽어와 해당 대시보드로 자동 리다이렉트.



\#### ② 교사(Teacher) 모드: AI 콘텐츠 허브

\*   \*\*기능:\*\* PDF 교안 업로드 → AI가 핵심 키워드 추출 및 \*\*객관식 퀴즈 자동 생성\*\*.

\*   \*\*UI:\*\* 파일 업로드 섹션 + 실시간 생성된 퀴즈 미리보기 및 편집 창.



\#### ③ 학생(Student) 모드: 개인 맞춤형 AI 튜터

\*   \*\*기능:\*\* 강의 자료 기반 실시간 Q\&A. "이 공식이 왜 이렇게 돼?"라고 물으면 AI가 \*\*교안의 관련 페이지\*\*를 인용하며 설명.

\*   \*\*UI:\*\* 좌측(강의 자료) / 우측(AI 챗봇) 분할 화면.



\#### ④ 운영자(Operator) 모드: 경영 인사이트 및 슬롯 관리

\*   \*\*기능:\*\* 현재 가입된 학생 수 실시간 모니터링 (`max\_slots` vs `current\_users`).

\*   \*\*비즈니스 시뮬레이션:\*\* "결제하기" 클릭 시 슬롯이 즉시 증가하고 기능이 확장되는 모습 시연.

\*   \*\*UI:\*\* 주요 지표 카드(Metric Cards) + 질문 빈도 통계 그래프.
