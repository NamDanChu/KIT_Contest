## 2. 완료된 내용 (코드 기준, 2026-04)

### 2.1 공통·인프라

- 앱 루트: **`EduChain_AI/`** — `streamlit run Home.py`
- 멀티페이지: `Home.py`, `pages/1_Login.py`, `2_관리.py`, `3_Teacher.py`, `4_학생관리.py`, `5_Student.py`
- **requirements.txt**: `streamlit>=1.33` 권장(`st.fragment`, `st.dialog` 등)
- Firebase Admin (`services/firebase_app.py`), Firestore (`firestore_repo.py`), 세션 키 (`session_keys.py`)

### 2.2 Firestore 데이터 (핵심만)

- `Organizations`, `Users`, `ChatLogs`
- `ContentCategories` — `teacher_uids` / `student_uids`, **`operator_feedback_teacher`**, **`operator_feedback_student`**
- `LessonWeeks` — 주차 메타·영상 URL·라이브·공개 설정·퀴즈 문항(학생/교사 화면에서 `quiz_session_params`로 사용)
- `Users/{uid}/LessonProgress` — **시청 %**, **`last_video_position_sec`**, **`video_duration_sec`**, **퀴즈**: `quiz_correct` / `quiz_total`, `quiz_passed`, **`quiz_attempt_count`**, **`quiz_wrong_indices`**, `quiz_wrong_count`
- `StudentLessonQuestions` — 질문·답변·**`video_position_sec`**, **`video_position_label`**
- **`aggregate_quiz_stats_for_course`** — 배정 학생×주차별 퀴즈 응시·제출·오답 합·문항별 오답 빈도 집계

### 2.3 학생 (`5_Student.py` + `student_portal.py`)

- 사이드바: 개요, 수업 선택, 수업 개요 / 수업 수강
- 수강: 주차 목록 → 플레이어(왼쪽 영상, 오른쪽 패널)
- 영상: YouTube/Vimeo/HTML5, 진행률·**재생 위치** 주기 저장(가능 시 Firebase 웹 설정)
- 우측: 라이브(데모)·AI 채팅·주차 개요
- **수업 개요**: 운영자 **학생 공개 안내** 표시
- **퀴즈**: 제출 시 `merge_student_lesson_quiz_result`로 채점·**오답 문항 인덱스**·**응시 횟수** 누적 저장

### 2.4 교사 (`3_Teacher.py` + `lesson_mgmt_ui.py` + `course_stats_ui.py`)

- 사이드바: 개요, 수업 선택, **학생 관리**, **수업 통계**, **AI 토큰 활용량**, **수업 관리**
- 개요: **운영자 피드백**, 학생 AI 질문(영상 위치), 주차 바로가기
- 학생 관리: 학생별 상세·주차 질문·**주차별 퀴즈 요약·오답 지문(가능 시)**·**학생 AI 분석**(`analyze_student_learning_profile`에 **`quiz_summary_block`** 전달, 결과 **expander**)
- 수업 통계: 주차 카드·질문·**퀴즈 집계(전체·선택 주차·문항 오답 빈도)**·**수업 전반 AI 분석**(`quiz_block` 반영, 결과 **expander**)
- 수업 관리: 주차 CRUD, 데이터 피드백 루프, Gemini 재출제 시 덮어쓰기 동의 등

### 2.5 운영자 (`2_관리.py` + `mgmt_*`)

- 기업·플랜·슬롯·사용자
- **콘텐츠**: 카테고리 이름·교사 배치만 편집(설명·하위 JSON 입력란 제거, DB 기존 필드는 유지)
- 과목 expander 안: **과목 통계·운영 피드백** — 교사 통계와 동일 + **AI 교사 피드백 초안** + 저장

### 2.6 인증·가입·승인 (2026-04 기준)

- **회원가입**: **가입 유형** — 운영자 / 유저(학원명 입력 없음). 유저는 **Home**에서 초대 코드로 **소속 신청**만 하고, **운영자 승인** 후 `Teacher`/`Student` 전환.
- **초대 코드 가입·Home 소속 신청**: 즉시 소속되지 않고 `Organizations/{org_id}/JoinRequests/{uid}` + 사용자 `membership_pending` 등으로 **승인 대기** → `mgmt_people` **가입 승인 대기**에서 승인/거절.
- **비밀번호**: 일반·초대 가입 모두 **비밀번호 확인** 입력·일치 검증, 6자 이상.
- **Home**: `User` 역할은 승인 대기 안내·초대 입력; 로그인 후 Firestore 프로필 **동기화**(`refresh_session_from_firestore`)로 승인 직후 역할 반영.

### 2.7 교사·학생 — 통합 퀴즈·로그 (요약)

- `services/student_quiz_mix.py`: 통합 퀴즈 **일괄 / 무한 연습**, 무한 모드는 **AI 코칭 요약 버튼**(자동 5문항 갱신 제거), 내부 ID 비노출 통계 문자열.
- `Organizations/.../StudentIntegratedQuizLogs`: 통합 퀴즈 활동 이벤트 저장 → `3_Teacher` 학생 패널에서 **통합 퀴즈(연습) 기록** 조회.

### 2.8 운영자 — 교사·학생 탭 UX (`mgmt_people.py`)

- **소속 사용자 목록**: 검색·정렬·스크롤 표(`st.dataframe`), **행 선택 시** 하단 「상세 정보를 볼 계정」과 동기화.
- **선택한 사용자 정보**: 보기 카드·UID 마스킹, **정보 변경**(닉네임·역할·비밀번호)·**계정 삭제**(기본 정보 탭과 동일 검증·슬롯).

### 2.9 AI 토큰 활용량 · 미터링 (2026-04)

- **Firestore**: `AiTokenRollup` — 버킷(`teacher_lesson`, `student_chat`, …) + 세부 기능 `kind_*` 필드 누적; `AiTokenEvents` — 호출 1건 로그(`usage_kind`, `actor_*`, 토큰, 수업 `category_id`).
- **`gemini_client.py`**: `_generate` 성공 시 `usage_metadata` 기반 토큰 수 기록; 호출부에서 `usage`(조직·수업·버킷·`usage_kind`) 전달.
- **UI** (`services/ai_usage_ui.py`): 교사 **AI 토큰 활용량** 메뉴 / 운영 **AI 토큰 활용량** 탭 — 표·차트( **matplotlib** 가로 막대, 없으면 `st.bar_chart` 폴백), **사용자 검색**(이름·역할·UID), 최근 호출 표.
- **의존성**: `requirements.txt`에 `matplotlib>=3.8.0` — 배포 시 `pip install -r requirements.txt` 필요.
- **차트 한글(배포)**: 서버에 한글 폰트가 없을 때 축·범례가 □로 깨지는 문제 → **번들 나눔고딕 TTF**로 해결(상단 요약·`EduChain_AI_전체정리.md` §5.3).
- **참고**: 배포 이전 호출은 `actor_*` 미기록·세부 `kind_*` 없을 수 있음.

### 2.10 성능·Firestore·UI (2026-04)

- **`firestore_repo`**: `list_content_categories_for_teacher` / `for_student` — 전체 스트림 대신 **`teacher_uids` / `student_uids`의 `array_contains`** 쿼리; **`@st.cache_data(ttl=45)`**; 카테고리 생성·수정·삭제 시 **`invalidate_teacher_student_category_caches()`**.
- **`pages/3_Teacher.py` · `5_Student.py` + `sidebar_helpers`**: 한 rerun에서 목록을 **두 번 읽지 않도록** `categories` / `courses` 인자로 공유.
- **`auth_session.refresh_session_from_firestore`**: Home 등에서 **과도한 Firestore 읽기 방지**를 위해 **약 20초 간격 스로틀**.
- **`requirements.txt`**: 미사용 **`chromadb` 제거**.

## 3. 부분 완료 / 기술 부채

| 항목 | 설명 |
|------|------|
| **Firestore 보안 규칙** | Admin SDK는 규칙 우회. 클라이언트 직접 접근 시 **`org_id`·역할** 기준 강화 필요 |
| **영상 플레이어 CSS** | `stHorizontalBlock:nth-of-type(2)` 등 Streamlit DOM 의존 — 버전에 따라 열 스타일 깨질 수 있음 |
| **라이브 채팅** | UI·로컬 데모 수준, 강사 실시간 연동 없음 |
| **RAG vs 수강 AI** | 수강 AI는 주차 필드 기반; **PDF/Chroma 풀 RAG**와의 단일 플로우는 미정리 |
| **Chroma** | 클라우드 배포 시 비영속 볼륨 이슈(현재 패키지 미포함·스텁 위주) |
| **퀴즈 데이터 레거시** | 퀴즈 필드 도입 이전 제출만 있는 문서는 `quiz_attempt_count`가 0으로 보일 수 있음 → 재제출 또는 일괄 보정 스크립트 검토 |
| **퀴즈 지문 표시** | `LessonWeeks`에 해당 주차 퀴즈 문항이 없거나 편집 불일치 시 교사 **학생 관리**에서 “지문 미수록”만 표시됨 |

## 4. 미착수·우선 보완 (여기서부터 진행)

### 4.1 단기 (안정화·시연)

1. **실제 브라우저 시나리오 테스트** — 수강 → 퀴즈 제출 → 교사 수업 통계·학생 관리·AI 분석에 수치·문구가 일치하는지 확인
2. **Firestore 규칙** — `EduChain_AI/firebase/firestore.rules`에 `ContentCategories`(읽기)·`StudentLessonQuestions`(학생 생성·교사/운영자 읽기)·`LessonProgress` 반영. 콘솔에 **배포** 후 클라이언트 SDK로 검증
3. **Streamlit Cloud** — Secrets(`GEMINI`, Firebase, `FIREBASE_SERVICE_ACCOUNT_JSON` 등), OAuth 리디렉션 — `연동필요.md` §4.1 참고 (본 문서와 같은 `1.기획/` 폴더)
4. **퀴즈 UX** — 풀이 화면 **「정답 확인」** = 즉시 채점·서버 저장. 결과 화면 **「정답 보기」·「재시도」** 직전에 레거시 `quiz_attempt_count==0` 이면 동기화 merge(`_sync_quiz_attempt_if_needed`) 후 진행

### 4.2 중기 (기획과의 갭 메우기)

1. **교사 업로드 → Chroma → 학생 질문** 한 줄로 이어지는 시연(또는 주차별 첨부 PDF와 RAG 명확화)
2. ~~**통계 고도화** — 기간 필터, CSV 내보내기, 운영자 **기업 단위** 요약 카드~~ → **반영됨**: 수업 통계에서 질문 **기간 필터·CSV**, 운영자 콘텐츠 탭 상단 **기업 요약 메트릭** (`summarize_org_learning_snapshot`)
3. ~~**에러·빈 상태 UX**~~ → **반영됨**: `services/ui_messages.py`로 Gemini 키·소속 기업·배정 없음·영상 미등록 등 문구 통일(교사/학생 페이지·사이드바·수강 영상)
4. **운영자 피드백** — 교사에게 읽음 확인·히스토리(선택)
5. **퀴즈 분석 심화** — 시도별 이력(현재는 마지막 제출 스냅샷 중심)이 필요하면 별도 서브컬렉션·로그 설계

### 4.3 장기 (상용에 가깝게)

1. 라이브 세션 **실시간 백엔드**(WebSocket/Firestore 실시간 등)
2. 과금·슬롯 **강제** (가입/수강 시 검증)
3. 감사 로그·개인정보 보관 정책

### 4.4 비즈니스·과금 (다음 논의 주제)

현재 `Organizations`는 **`max_slots`(학생 수)**·**`plan`** 중심이다. **학생 수만으로는 변동비(Gemini 등)를 설명·회수하기 어렵다**는 한계가 있어, 아래 축을 기획·설계 단계에서 정리할 예정이다. (상세 프레임은 `EduChain_AI_전체정리.md` §5.1 참고)

| 축 | 예시 |
|----|------|
| **AI 사용량** | 월별 요청 수, 추정 토큰(입력+출력), 기능별(채팅·퀴즈 생성·통계 분석·통합 퀴즈 코칭) 가중치 — **UI·Firestore 집계는 반영됨**(과금·청구 연동은 미연결) |
| **모델 티어** | Flash(기본) vs Pro(고급 분석 옵션) — 플랜 또는 조직 단위 스위치 |
| **공정 사용 정책** | 플랜별 월 상한, 초과 시 알림·소프트 캡·유료 업셀 |
| **구현** | Firestore **`AiTokenRollup` / `AiTokenEvents`** 로 호출·토큰·기능·사용자 단위 **관측** 가능 — 일별/월별 **과금용** 집계·청구 API 연동은 추후 |
