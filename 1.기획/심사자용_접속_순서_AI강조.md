# 심사자용 접속 순서 (AI 기능 중심)

> 이 문서는 **메뉴 소개**보다, 심사 관점에서 중요한 **AI 구현 확인 포인트**를 빠르게 검증할 수 있도록 정리한 가이드입니다.

---

## 사전 안내 (필독)

- **라이브 URL:** `https://ndc-educhain-ai.streamlit.app/` *(배포 주소 — 필요 시 교체)*
- **첫 로딩:** Streamlit Cloud 특성상 유휴 후 첫 접속은 **30초~1분** 걸릴 수 있습니다.
- **역할별 메뉴 분리:** 로그인 후 역할(Operator / Teacher / Student)에 따라 좌측 메뉴가 다릅니다.
- **역할 전환 방법:** 다른 역할 화면을 확인하려면 **로그아웃 후 재로그인**하세요.
- **데이터 의존 화면:** 배정/로그가 없는 경우 일부 표·차트는 비어 보일 수 있습니다.

## 핵심 AI 기능 하이라이트 (심사 포인트)

본 서비스는 단순 LMS가 아니라, **LLM(Gemini) 기반 학습 보조 루프**를 구현한 것이 핵심입니다.

1. **교사 AI 제작 도구**
   - 수업 자료(교안 PDF, 자막 txt/md, 목표) 기반 **요약·키워드 자동 생성**
   - **객관식 퀴즈 풀(JSON) 자동 생성·저장** 및 학생용 출제 연결
   - 마크다운 퀴즈/TXT, 한 페이지 요약 노트 생성
   - 위치: `Teacher > 수업 관리`

2. **학생 AI 학습 보조**
   - 주차 맥락(학습목표·요약·키워드)에 근거한 **AI 튜터 Q&A**
   - 통합 퀴즈 결과 기반 **AI 총평·해설·보완점**, 무한 연습 **AI 코칭 요약**
   - 위치: `Student > 수업 수강`, `Student > 통합 퀴즈`

3. **교사/운영 AI 분석**
   - 학생별 학습 패턴 분석(질문·진행률·퀴즈 요약 기반)
   - 수업 전반 분석(주차 완료율·질문·퀴즈 집계 기반)
   - 운영자 피드백 초안 자동 작성
   - 위치: `Teacher > 학생 관리`, `Teacher > 수업 통계`, `Operator > 관리 > 콘텐츠·통계`

4. **AI 호출량 추적(실사용 검증 지표)**
   - 호출 1건 단위 이벤트 + 누적 롤업으로 **입력/출력 토큰, 기능별 사용량, 사용자별 사용량** 시각화
   - 위치: `Teacher > AI 토큰 활용량`, `Operator > 관리 > AI 토큰 활용량`

## 추천 확인 순서 (AI 시나리오 중심)

아래 순서로 보면 "교사가 AI로 만들고 -> 학생이 학습/질문/퀴즈를 수행하고 -> 교사/운영이 AI 분석으로 피드백하는 흐름"이 명확히 보입니다.

### 1) Teacher: AI 콘텐츠 생성

1. 교사 계정 로그인 -> `Teacher` 진입
2. `수업 관리`에서 수업/주차 선택
3. **AI 지식 학습 시작 (Gemini 요약·키워드)** 실행
4. **Gemini로 객관식 문항 생성·저장** 실행
5. (선택) `주차별 퀴즈 생성`, `한 페이지 요약 노트` 실행

> 확인 포인트: 생성 결과가 주차 데이터에 저장되고, 학생 수강/퀴즈 화면에 연결됩니다.

### 2) Student: AI 튜터링 + 통합 퀴즈

1. 학생 계정 로그인 -> `Student` 진입
2. `수업 수강`에서 주차 선택 후 우측 패널 **AI 채팅**에서 질문
3. `통합 퀴즈` 진입 후 일괄/무한 연습 수행
4. 제출 후 **AI 총평·해설·보완점**, **AI 코칭 요약** 확인

> 확인 포인트: 학생 질문/응답, 퀴즈 결과가 이후 교사 분석과 토큰 집계에 반영됩니다.

### 3) Teacher: AI 분석 확인

1. 교사 계정으로 `학생 관리` 진입
2. 학생 선택 후 **학생 분석하기** 실행
3. `수업 통계`에서 **수업 통계 분석하기** 실행

> 확인 포인트: 질문/진행률/퀴즈 요약 데이터를 기반으로 한 분석 결과가 출력됩니다.

### 4) Operator: 운영 관점 AI 검증

1. 운영자 계정 로그인 -> `관리` 진입
2. 기업 선택 -> `콘텐츠·통계`에서 과목 통계/AI 분석 확인
3. **AI로 교사 피드백 초안 작성** 실행
4. `AI 토큰 활용량` 탭에서 기업/수업/사용자 단위 사용량 점검

> 확인 포인트: 운영자 관점에서도 AI 분석과 피드백 루프가 동작합니다.

### 5) 공통: AI 토큰 활용량 검증

1. `Teacher > AI 토큰 활용량`에서 수업 단위 용도·세부기능·사용자·최근호출 확인
2. `Operator > 관리 > AI 토큰 활용량`에서 기업 전체 집계 및 수업별 비교 확인

> 핵심 메시지: "실제로 AI가 호출되어 비용/리소스가 추적되는 서비스"임을 수치로 증명합니다.

## 역할별 상세 확인 체크리스트

### A. 비로그인
- `Home.py` 진입 확인 (브랜딩/기본 안내)
- `Login` 이동 확인

### B. 운영자 (Operator)
- `관리` 접근 가능 여부
- 기업 선택 후 카테고리 이동:
  - `기본 정보` -> `플랜·슬롯` -> `교사·학생` -> `콘텐츠·통계` -> `AI 토큰 활용량`
- `AI 토큰 활용량` 차트/표 노출 확인

### C. 교사 (Teacher)
- `Teacher` 접근 가능 여부
- `개요 / 학생 관리 / 수업 통계 / AI 토큰 활용량 / 수업 관리` 메뉴 노출 확인
- `수업 관리`에서 Gemini 기능 동작 확인

### D. 학생 (Student)
- `Student` 접근 가능 여부
- `수업 개요 / 수업 수강 / 통합 퀴즈` 전환 확인
- `수업 수강` AI 채팅, `통합 퀴즈` AI 해설 노출 확인

## 코드 기준 참고 (심사용)

### 페이지 진입
- `EduChain_AI/Home.py` — 홈/역할 안내
- `EduChain_AI/pages/1_Login.py` — 로그인/회원가입
- `EduChain_AI/pages/2_관리.py` — 운영자
- `EduChain_AI/pages/3_Teacher.py` — 교사
- `EduChain_AI/pages/5_Student.py` — 학생

### AI 핵심 로직
- `EduChain_AI/services/gemini_client.py`
  - 요약/키워드, 퀴즈 생성(JSON/MD), 학생 Q&A, 학생/수업 분석, 운영자 피드백 초안
- `EduChain_AI/services/lesson_mgmt_ui.py`
  - 교사 수업 관리의 AI 실행 UI(자료 학습, 퀴즈 생성/저장, 노트/키워드)
- `EduChain_AI/services/student_portal.py`
  - 학생 수강 AI 채팅, 주차별 수강/퀴즈 연동
- `EduChain_AI/services/student_quiz_mix.py`
  - 통합 퀴즈 AI 해설/코칭
- `EduChain_AI/services/course_stats_ui.py`
  - 교사·운영 수업 통계 AI 분석, 운영자 피드백 초안

### AI 사용량(미터링) 로직
- `EduChain_AI/services/ai_usage_ui.py` — 토큰 활용량 표/차트/최근호출 UI
- `EduChain_AI/services/firestore_repo.py`
  - `increment_ai_token_rollup`, `append_ai_token_event`
- `EduChain_AI/services/sidebar_helpers.py` — 역할별 메뉴 노출/전환
