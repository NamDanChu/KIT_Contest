## 2. 완료된 내용 (코드 기준, 2026-04)

### 2.1 공통·인프라

- 앱 루트: **`EduChain_AI/`** — `streamlit run Home.py`
- 멀티페이지: `Home.py`, `pages/1_Login.py`, `2_관리.py`, `3_Teacher.py`, `4_학생관리.py`, `5_Student.py`
- **requirements.txt**: `streamlit>=1.33` 권장(`st.fragment`, `st.dialog` 등)
- Firebase Admin (`services/firebase_app.py`), Firestore (`firestore_repo.py`), 세션 키 (`session_keys.py`)

### 2.2 Firestore 데이터 (핵심만)

- `Organizations`, `Users`, `ChatLogs`
- `ContentCategories` — `teacher_uids` / `student_uids`, **`operator_feedback_teacher`**, **`operator_feedback_student`**
- `LessonWeeks` — 주차 메타·영상 URL·라이브·공개 설정·퀴즈 문항(학생/교사 화면에서 `quiz_session_params`로 사용)
- `Users/{uid}/LessonProgress` — **시청 %**, **`last_video_position_sec`**, **`video_duration_sec`**, **퀴즈**: `quiz_correct` / `quiz_total`, `quiz_passed`, **`quiz_attempt_count`**, **`quiz_wrong_indices`**, `quiz_wrong_count`
- `StudentLessonQuestions` — 질문·답변·**`video_position_sec`**, **`video_position_label`**
- **`aggregate_quiz_stats_for_course`** — 배정 학생×주차별 퀴즈 응시·제출·오답 합·문항별 오답 빈도 집계

### 2.3 학생 (`5_Student.py` + `student_portal.py`)

- 사이드바: 개요, 수업 선택, 수업 개요 / 수업 수강
- 수강: 주차 목록 → 플레이어(왼쪽 영상, 오른쪽 패널)
- 영상: YouTube/Vimeo/HTML5, 진행률·**재생 위치** 주기 저장(가능 시 Firebase 웹 설정)
- 우측: 라이브(데모)·AI 채팅·주차 개요
- **수업 개요**: 운영자 **학생 공개 안내** 표시
- **퀴즈**: 제출 시 `merge_student_lesson_quiz_result`로 채점·**오답 문항 인덱스**·**응시 횟수** 누적 저장

### 2.4 교사 (`3_Teacher.py` + `lesson_mgmt_ui.py` + `course_stats_ui.py`)

- 사이드바: 개요, 수업 선택, **학생 관리**, **수업 통계**, **수업 관리**
- 개요: **운영자 피드백**, 학생 AI 질문(영상 위치), 주차 바로가기
- 학생 관리: 학생별 상세·주차 질문·**주차별 퀴즈 요약·오답 지문(가능 시)**·**학생 AI 분석**(`analyze_student_learning_profile`에 **`quiz_summary_block`** 전달, 결과 **expander**)
- 수업 통계: 주차 카드·질문·**퀴즈 집계(전체·선택 주차·문항 오답 빈도)**·**수업 전반 AI 분석**(`quiz_block` 반영, 결과 **expander**)
- 수업 관리: 주차 CRUD, 데이터 피드백 루프, Gemini 재출제 시 덮어쓰기 동의 등

### 2.5 운영자 (`2_관리.py` + `mgmt_*`)

- 기업·플랜·슬롯·사용자
- **콘텐츠**: 카테고리 이름·교사 배치만 편집(설명·하위 JSON 입력란 제거, DB 기존 필드는 유지)
- 과목 expander 안: **과목 통계·운영 피드백** — 교사 통계와 동일 + **AI 교사 피드백 초안** + 저장

## 3. 부분 완료 / 기술 부채

| 항목 | 설명 |
|------|------|
| **Firestore 보안 규칙** | Admin SDK는 규칙 우회. 클라이언트 직접 접근 시 **`org_id`·역할** 기준 강화 필요 |
| **영상 플레이어 CSS** | `stHorizontalBlock:nth-of-type(2)` 등 Streamlit DOM 의존 — 버전에 따라 열 스타일 깨질 수 있음 |
| **라이브 채팅** | UI·로컬 데모 수준, 강사 실시간 연동 없음 |
| **RAG vs 수강 AI** | 수강 AI는 주차 필드 기반; **PDF/Chroma 풀 RAG**와의 단일 플로우는 미정리 |
| **Chroma** | 클라우드 배포 시 비영속 볼륨 이슈 |
| **퀴즈 데이터 레거시** | 퀴즈 필드 도입 이전 제출만 있는 문서는 `quiz_attempt_count`가 0으로 보일 수 있음 → 재제출 또는 일괄 보정 스크립트 검토 |
| **퀴즈 지문 표시** | `LessonWeeks`에 해당 주차 퀴즈 문항이 없거나 편집 불일치 시 교사 **학생 관리**에서 “지문 미수록”만 표시됨 |

## 4. 미착수·우선 보완 (여기서부터 진행)

### 4.1 단기 (안정화·시연)

1. **실제 브라우저 시나리오 테스트** — 수강 → 퀴즈 제출 → 교사 수업 통계·학생 관리·AI 분석에 수치·문구가 일치하는지 확인
2. **Firestore 규칙** — `EduChain_AI/firebase/firestore.rules`에 `ContentCategories`(읽기)·`StudentLessonQuestions`(학생 생성·교사/운영자 읽기)·`LessonProgress` 반영. 콘솔에 **배포** 후 클라이언트 SDK로 검증
3. **Streamlit Cloud** — Secrets(`GEMINI`, Firebase), OAuth 리디렉션 — `기획/연동필요.md` §4.1 참고
4. **퀴즈 UX** — 풀이 화면 **「정답 확인」** = 즉시 채점·서버 저장. 결과 화면 **「정답 보기」·「재시도」** 직전에 레거시 `quiz_attempt_count==0` 이면 동기화 merge(`_sync_quiz_attempt_if_needed`) 후 진행

### 4.2 중기 (기획과의 갭 메우기)

1. **교사 업로드 → Chroma → 학생 질문** 한 줄로 이어지는 시연(또는 주차별 첨부 PDF와 RAG 명확화)
2. **통계 고도화** — 기간 필터, CSV 내보내기, 운영자 **기업 단위** 요약 카드
3. **에러·빈 상태 UX** — API 키 없음, 영상 없음, 배정 없음 등 메시지 통일
4. **운영자 피드백** — 교사에게 읽음 확인·히스토리(선택)
5. **퀴즈 분석 심화** — 시도별 이력(현재는 마지막 제출 스냅샷 중심)이 필요하면 별도 서브컬렉션·로그 설계

### 4.3 장기 (상용에 가깝게)

1. 라이브 세션 **실시간 백엔드**(WebSocket/Firestore 실시간 등)
2. 과금·슬롯 **강제** (가입/수강 시 검증)
3. 감사 로그·개인정보 보관 정책
