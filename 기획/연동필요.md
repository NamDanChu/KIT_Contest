# EduChain AI — 연동에 필요한 키·계정 정리

이 프로젝트는 **로컬 개발**과 **Streamlit Cloud 배포**에서 비밀 값을 씁니다. 아래 순서대로 준비하면 됩니다.

---

## 1. 준비물 요약

| 구분 | 무엇이 필요한가 | 주로 쓰는 위치 |
|------|-----------------|----------------|
| **Google Gemini** | API 키 1개 | `GEMINI_API_KEY` — 퀴즈·RAG 답변 |
| **Firebase** | 프로젝트 + (선택) 서비스 계정 JSON | Auth, Firestore — 로그인·DB |
| **Streamlit Cloud** | (배포 시) 위 키들을 대시보드에 등록 | GitHub 연동 앱의 Secrets |

**원칙:** 키·JSON은 **Git에 커밋하지 않습니다.** 로컬은 `EduChain_AI/.streamlit/secrets.toml`, 배포는 Streamlit Cloud **Secrets**에만 넣습니다.

---

## 2. Google Gemini API 키

### 왜 필요한가

- PDF 분석, 퀴즈 생성, 학생 질문 답변(RAG)에 **Gemini API**를 쓰기 때문입니다.

### 어떻게 가져오는가

1. Google 계정으로 [Google AI Studio](https://aistudio.google.com/)에 접속합니다.
2. 왼쪽 또는 상단 메뉴에서 **Get API key** / **API 키** 로 이동합니다.
3. **Create API key** 로 키를 생성합니다.  
   - 프로젝트를 묶고 싶으면 Google Cloud 프로젝트를 선택하거나 새로 만듭니다.
4. 생성된 문자열을 복사해 둡니다. **다시 전체 표시되지 않을 수 있으므로** 안전한 곳에 보관합니다.

### 로컬에 넣는 방법

1. `EduChain_AI/.streamlit/secrets.toml.example` 을 복사해 `secrets.toml` 로 저장합니다.
2. 파일 안에 다음처럼 넣습니다.

```toml
GEMINI_API_KEY = "여기에_붙여넣기"
```

3. 코드에서 `st.secrets["GEMINI_API_KEY"]` 로 읽거나, 환경 변수 `GEMINI_API_KEY` 로도 줄 수 있습니다(터미널 세션 한정).

### 참고

- 할당량·요금은 [Google AI 개발자 문서](https://ai.google.dev/)의 정책을 따릅니다. 공모전/데모는 무료 한도 안에서 쓰는 경우가 많습니다.
- 키가 유출되면 AI Studio에서 **키를 삭제하고 새로 발급**합니다.

## 3. Firebase (Auth + Firestore)

### 왜 필요한가

- **이메일/비밀번호 로그인**, 사용자·조직·채팅 로그 저장에 **Firebase Authentication**과 **Cloud Firestore**를 쓰기 때문입니다.

### 어떻게 가져오는가 (콘솔에서)

1. [Firebase Console](https://console.firebase.google.com/) 에 접속합니다.
2. **프로젝트 추가**로 새 프로젝트를 만듭니다(이름은 예: `educhain-ai`).
3. **Authentication** 메뉴 → **시작하기** → 로그인 방법에서 **이메일/비밀번호** 를 켭니다.
4. **Firestore Database** 메뉤 → **데이터베이스 만들기** → 보안 규칙은 개발 단계에서는 테스트 모드로 시작할 수 있으나, **실제 서비스/제출 전에는 규칙을 반드시 강화**합니다.
5. (웹/클라이언트 SDK를 쓸 경우) 프로젝트 설정(톱니바퀴) → **일반** 탭에서 필요 시 **내 앱**에 웹 앱을 등록해 `apiKey`, `authDomain`, `projectId` 등 **웹 설정 객체**를 확인합니다.

### 서버(Streamlit·`firebase-admin`)용 서비스 계정 JSON

Python에서 **firebase-admin** 으로 Firestore에 접근하려면 **서비스 계정 키**가 필요합니다.

1. Firebase Console → **프로젝트 설정** → **서비스 계정** 탭.
2. **새 비공개 키 생성** 으로 JSON 파일을 다운로드합니다.
3. 파일을 예: `EduChain_AI/firebase-service-account.json` 처럼 프로젝트 폴더에 두고, **`.gitignore`에 이미 제외되어 있는지 확인**합니다. 저장소에는 절대 올리지 않습니다.
4. 앱 초기화 시 이 JSON 경로를 환경 변수나 `secrets.toml`에 **파일 경로 문자열**로 넣는 방식이 일반적입니다. (JSON 내용 전체를 secrets에 붙여 넣는 방식도 가능하나 관리가 번거롭습니다.)

`secrets.toml` 예시:

```toml
GEMINI_API_KEY = "..."

# firebase-admin 초기화용 (파일 경로 또는 환경 변수명은 코드와 맞출 것)
FIREBASE_CREDENTIALS_PATH = "firebase-service-account.json"
```

또는 Streamlit Cloud에는 JSON 파일을 직접 올릴 수 없으므로, **JSON 내용을 Base64로 넣거나**, Cloud Secret에 **여 줄 문자열**로 넣어 코드에서 파싱하는 방식을 쓰는 팀도 있습니다. **가장 단순한 로컬 개발**은 위처럼 JSON 파일을 로컬에만 두는 방법입니다.

## 4. Streamlit Cloud에 넣는 방법 (배포 시)

1. [Streamlit Cloud](https://streamlit.io/cloud) 에서 GitHub 저장소를 연결합니다.
2. 앱 설정 → **Secrets** 에 `secrets.toml` 과 **동일한 키 이름**으로 값을 넣습니다.

예:

```toml
GEMINI_API_KEY = "..."
```

Firebase는 배포 환경에 맞게 `FIREBASE_...` 키들을 맞춰 넣고, 앱 코드에서 `st.secrets` 로 읽도록 통일합니다.
