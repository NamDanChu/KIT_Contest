## 1. 권장 개발 순서 (의존 관계 순)

### 1단계: Firebase 백엔드와 앱 연결 — 완료

구현 위치: `EduChain_AI/services/firebase_app.py`, `EduChain_AI/services/firestore_repo.py`, `EduChain_AI/firebase/firestore.rules`

- [x] **`firebase-admin` 초기화** (`services/firebase_app.py`)  
  - 환경 변수 `FIREBASE_SERVICE_ACCOUNT_JSON` / `FIREBASE_CREDENTIALS_PATH` / `GOOGLE_APPLICATION_CREDENTIALS`  
  - Streamlit 실행 시 `st.secrets` 동일 키 지원  

- [x] **Firestore 컬렉션·필드** (코드 상 상수 및 `set`/`get` 필드명)  
  - `Organizations`, `Users`, `ChatLogs` — 기획서와 정렬  

- [x] **보안 규칙 초안**  
  - `firebase/firestore.rules` — 콘솔에 배포해 적용. Admin SDK는 규칙 우회(서버 전용). 제출 전 `org_id` 기준으로 강화  

- [x] **`firestore_repo.py` 구현**  
  - 조직·사용자 조회/저장, 채팅 로그 append, org별 사용자 수  

**완료 기준:** 로컬에서 자격 증명 설정 후 `firestore_repo` 함수로 읽기·쓰기 가능(콘솔에서도 문서 확인).

---

### 2단계: 인증·세션·역할 분기 (게이트웨이)

1. **`firebase_auth.py` (또는 Auth 어댑터)**  
   - 이메일/비밀번호 로그인·회원가입·로그아웃  
   - Streamlit에서 쓰기 쉬운 방식으로 **uid** 확보  

2. **`session_keys` + `st.session_state`**  
   - `auth_uid`, `auth_role`, `auth_org_id` 유지  

3. **`pages/1_Login.py`**  
   - 폼 연결, 성공 시 Firestore `Users`에서 `role` 읽기  

4. **역할별 진입 제어**  
   - 미로그인 시 민감 페이지에서 `st.switch_page` 또는 안내 후 로그인 유도  

**완료 기준:** 로그인 후 역할에 따라 Teacher / Student / Operator 화면으로 분기 가능.

---

### 3단계: AI 코어 — Gemini + RAG

1. **`gemini_client.py`**  
   - `st.secrets["GEMINI_API_KEY"]` 사용  
   - 퀴즈 생성·질문 답변 함수 구현  

2. **`rag_pipeline.py`**  
   - PDF 텍스트 추출 (`pypdf` 등)  
   - 청킹·임베딩·ChromaDB upsert / query  
   - `org_id` 메타데이터로 **검색 필터**  

**완료 기준:** 로컬에서 PDF 업로드 → 질문 → 컨텍스트 기반 답변 한 사이클.

---

### 4단계: 역할별 화면 (MVP)

| 순서 | 페이지 | 내용 |
|------|--------|------|
| 1 | `2_Teacher.py` | 파일 업로드 → RAG 인입 → 퀴즈 생성 결과 표시 |
| 2 | `3_Student.py` | 좌우 레이아웃 + 채팅 → RAG 답변 → `ChatLogs` 저장 |
| 3 | `4_Operator.py` | `Organizations`·`Users` 집계, 슬롯 vs 인원, 간단 차트 |

**완료 기준:** 시연용 시나리오(교사 업로드 → 학생 질문 → 운영자 지표)가 끊기지 않게 동작.

---

### 5단계: UI·안정성·배포

1. **에러 메시지** — 사용자용 한글, API 실패 시 재시도 안내  
2. **테마·레이아웃** — `.streamlit/config.toml`과 필요 시 최소 CSS  
3. **GitHub 푸시 + Streamlit Cloud**  
   - Main file: `EduChain_AI/Home.py`  
   - Secrets에 로컬과 동일 키 이름 유지  
4. **클라우드 제약** — Chroma 비영속 시 “재업로드” 안내 또는 저장 전략 문서화  

---

### 6단계: 제출·발표

1. 기술 문서(보안·비즈니스) 정리  
2. 시연 영상·스크립트  
3. 제출물 목록 점검
