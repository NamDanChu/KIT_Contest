# 01. AI 협업 프로세스

## 목적

인간(기획·개발)과 **AI 보조 도구**가 같은 목표(EduChain AI)를 두고, **반복 가능한 방식**으로 기획을 구체화하고 코드·문서에 반영하는 과정을 설명합니다.

---

## 협업 단계 (순서)

1. **기획 수립**  
   - 문제 정의(교육 SaaS, Firebase, 역할 분리, AI 비용 가시화 등)를 사람이 먼저 잡고, `정리폴더/`, `기획/`에 Markdown으로 정리합니다.

2. **기획 ↔ 코드 정합성**  
   - `기획/EduChain_AI_전체정리.md`에 데이터 모델·화면·로드맵을 모으고, `기획/진행중상황.md`로 **완료/미완**을 주기적으로 맞춥니다.

3. **구현 시 AI 활용**  
   - Cursor 등에서 **구체적 맥락**(파일 경로, 재현 단계, 기대 동작)을 주고, 리팩터·버그 수정·최적화를 요청합니다.  
   - 결과는 **diff로 검토**하고, 필요 시 기획 문서·이 폴더의 **이력**에 남깁니다.

4. **문서 동기화**  
   - 기능 추가(예: AI 토큰 미터링, 성능 개선) 후 `기획/github용.md`, `진행중상황.md`, `EduChain_AI_전체정리.md`를 갱신해 **제3자(심사·동료)가 따라갈 수 있게** 합니다.

5. **제출 패키지**  
   - 본 `AI_협업_자료` 폴더로 **협업 방식과 문서 색인**을 한 번에 제시합니다.

---

## 효율적 협업을 위해 지킨 습관

- 요청은 **한 가지 목표**씩(예: “배포 시 차트 한글 깨짐만 해결”).
- **코드베이스 경로**(`EduChain_AI/services/…`)를 명시해 환각을 줄임.
- AI가 제안한 변경은 **기존 스타일·네이밍**과 맞는지 확인 후 머지.
